# RAG実画面確認チェックリスト（社内規程）

## 1. 事前準備

1. ターミナルでプロジェクトルートへ移動する。  
`cd /Users/tamuraakihiro/Desktop/ai-chatbot-app`
2. 仮想環境を有効化する。  
`source venv/bin/activate`
3. アプリを起動する。  
`streamlit run app.py`
4. ブラウザで `http://localhost:8501` を開く。
5. 新規会話を作成し、他のテキスト入力が残っていない状態にする。

## 2. 確認時の見方

1. 各入力文を送信後、回答内または回答直下の `📚 参照規程 (n件)` を開く。
2. 下記「期待ヒット」に記載した規程コードが1件以上含まれていれば合格。
3. `n件=0` の場合は不合格。

## 3. 入力文ごとの期待ヒット

| No | 入力文 | 期待ヒット（いずれか1件以上） | 補足 |
|---|---|---|---|
| 1 | 社内文書を参照して教えて | `SEC-001` または `WK-001` または `CMP-001` | 曖昧語（社内文書）でのヒット確認 |
| 2 | 社内規定を教えて | `SEC-001` または `WK-001` または `CMP-001` | 規定/規程の表記ゆれ確認 |
| 3 | 有給休暇の申請期限は？ | `HR-002 / 2.1` | 「有給申請期限」条項への到達確認 |
| 4 | 残業申請のルールは？ | `HR-001 / 1.3` | 「残業申請」条項への到達確認 |
| 5 | パスワード要件を教えて | `SEC-001 / 4.1` | 複合語（パスワード要件）分割確認 |
| 6 | 在宅勤務日の開始報告はいつまで？ | `WK-001 / 3.2` | 在宅勤務規程への到達確認 |
| 7 | 経費精算の申請期限は？ | `FN-001 / 6.1` | 経費精算規程への到達確認 |
| 8 | インシデントの報告期限は？ | `SEC-002 / 5.1` | インシデント規程への到達確認 |
| 9 | ハラスメント相談窓口は？ | `CMP-001 / 7.2` | コンプライアンス規程への到達確認 |
|10 | 生成AIに入力禁止の情報は？ | `IT-001 / 8.1` | 生成AIガイドラインへの到達確認 |

## 4. 失敗時の切り分け

1. `📚 参照規程` が常に0件の場合  
`python3 database/seed_demo_policies.py` を実行して再投入後、アプリ再起動。
2. 一部の入力文のみ外す場合  
表記を短くして再試行（例: 「パスワード要件」→「パスワード」）。
3. 期待外れの規程が上位になる場合  
`database/db_manager.py` の `_build_query_tokens` の重みを調整。

## 5. 合格基準

1. No.1〜No.10 の全入力で、期待ヒットが1件以上含まれること。
2. 10件中9件以下しか通らない場合は不合格として再調整する。
